SOC Mini SIEM Lab

Lightweight SOC (Security Operations Center) Mini SIEM Lab built in Bash on Linux/Termux.

This project demonstrates core SOC monitoring concepts including File Integrity Monitoring (FIM), Event Correlation, Burst Detection, Incident Logging, Log Rotation, and a real-time SOC Dashboard.

---

Features

File Integrity Monitoring (FIM)

Monitors files and detects:

- File Creation
- File Modification
- File Deletion

Event Correlation

Correlates monitored events and assigns severity levels:

- LOW
- MEDIUM
- CRITICAL

Burst Detection

Detects rapid repeated activity that may indicate:

- Automated attacks
- Malicious scripts
- Suspicious file activity

Incident Logging

Creates structured incident records including:

- Timestamp
- Incident ID
- Severity
- Event Type
- Target File
- Detection Score

SOC Dashboard

Provides real-time visibility into:

- Total Incidents
- Severity Breakdown
- Event Types
- Threat Status
- Recent Activity

Log Rotation

Archives historical incidents and prevents excessive log growth.

Installation

Clone the repository:

git clone https://github.com/thabosakonta-wq/soc-mini-siem-lab.git
cd soc-mini-siem-lab

Make scripts executable:

chmod +x *.sh
chmod +x engine/*.sh

---

Running the Lab

Start the SOC lab:

bash run_soc.sh

Launch the dashboard:

bash dashboard.sh

Detection Workflow

1. File activity occurs.
2. FIM Engine detects the change.
3. Correlation Engine evaluates the event.
4. Incident Correlator assigns severity.
5. Burst Detection checks event frequency.
6. Incident is written to the log.
7. Dashboard displays updated metrics.
